# Offer 捕手当前维护说明

> 更新日期：2026-06-14

## 当前真实运行入口

- 前端入口：`frontend/index.html`
- 后端入口：`backend/app/main.py`
- 一键启动：`./start.sh`
- 前端端口：`3000`
- 后端端口：`8888`

本项目当前不是 Next.js 项目。前端通过 `python3 -m http.server 3000` 启动，不需要 `npm install` 或 `npm run dev`。

## 历史文件处理原则

仓库中保留了一些历史文件和备份文件，例如：

- `frontend/index-new-backup.html`
- `backend/app/main.py.bak`
- 早期 TinyDB 数据层：`backend/app/db/database.py`
- 根目录和 `docs/` 下的早期方案文档

这些文件暂时不删除。后续如果要继续清理，必须先确认：

1. 当前主流程没有引用该文件。
2. 启动脚本和测试脚本不依赖该文件。
3. 已完成至少一次完整本地验证。
4. 清理前有可回退的备份或版本记录。

## 当前推荐开发边界

- 改前端：优先改 `frontend/index.html`。
- 改后端 API：优先改 `backend/app/main.py`，复用 `backend/app/services/` 下已有服务。
- 改启动方式：同步检查 `start.sh`、`start.bat` 和 `README.md`。
- 改文档：以真实代码、依赖文件和启动脚本为准，不把历史规划写成已完成事实。

## 下一步可清理清单

低风险：

- 持续修正文档中的旧端口、旧路径、旧技术栈描述。
- 把本地绝对路径改成相对路径或环境变量。
- 标注历史备份文件用途。

中风险：

- 确认 `backend/app/db/database.py` 无引用后再迁出或删除。
- 确认 `frontend/index-new-backup.html` 不再需要后再归档。
- 收敛根目录重复方案文档。

高风险：

- 拆分 `frontend/index.html`。
- 拆分 `backend/app/main.py`。
- 删除历史测试数据或数据库文件。

高风险项应单独开一轮任务，不建议和文档清理混在一起做。
