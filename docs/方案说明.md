# Offer 捕手 - 方案说明

## 一、问题诊断

### 用户痛点

学生在求职时面临三大核心痛点：

1. **信息过载**：海量岗位中难以找到匹配的机会
2. **匹配度未知**：不确定自己的简历与岗位的匹配程度
3. **优化无门**：不知道简历需要如何优化才能提升通过率

### 市场现状

- BOSS直聘、猎聘等平台主要做**岗位推荐**，不做**简历优化**
- 简历优化工具只做**简历优化**，不做**岗位匹配**
- 缺少一个**闭环解决方案**

## 二、方案设计

### 核心功能

```
简历上传 → AI解析 → 岗位输入 → JD解析 → 匹配计算 → 优化建议
```

### 技术架构

```
┌─────────────────────────────────────────────────────────┐
│              前端 (静态 HTML + 原生 JavaScript)             │
├─────────────────────────────────────────────────────────┤
│                   后端 (FastAPI + Python)                 │
├─────────────────────────────────────────────────────────┤
│                   AI引擎 (LiteLLM 多模型)                  │
├─────────────────────────────────────────────────────────┤
│                   数据存储 (SQLite)                       │
└─────────────────────────────────────────────────────────┘
```

## 三、AI 工具选型

### LLM 选择：豆包 API

**选型理由：**

| 维度 | 豆包 | 通义千问 | ChatGPT |
|------|------|----------|---------|
| 中文理解 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| 价格 | 0.008元/千tokens | 0.004元/千tokens | 0.15元/千tokens |
| 速度 | 快 | 快 | 慢 |
| 本土化 | 优秀 | 优秀 | 一般 |

**最终选择：豆包 API**
- 中文理解能力强
- 价格适中（性价比高）
- 服务稳定
- 支持 JSON 模式输出

### 技术框架选择

| 组件 | 技术选型 | 理由 |
|------|----------|------|
| 后端框架 | FastAPI | 高性能、自动API文档 |
| 前端框架 | 静态 HTML + 原生 JavaScript | 无需构建，适合当前单页原型快速迭代 |
| 样式方案 | Tailwind CDN + 自定义 CSS | 保持单文件前端的低启动成本 |
| LLM接口 | LiteLLM | 多模型统一接口 |

## 四、关键配置

### 匹配算法权重

```python
总匹配度 = 硬技能 × 40% + 软技能 × 20% +
          教育背景 × 15% + 经验匹配 × 15% +
          项目相关性 × 10%
```

### Prompt 设计原则

1. **Few-shot Learning**：提供示例帮助模型理解任务
2. **结构化输出**：使用 JSON 模式确保输出格式
3. **中文优化**：针对中文简历特点设计提示词
