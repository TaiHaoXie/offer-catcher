# Offer 捕手 - 智能求职匹配系统 方案说明

## 一、问题诊断

### 1.1 核心痛点

当前学生求职面临三大问题：

- **简历投递盲目**：一份简历投遍所有岗位，缺乏针对性优化
- **匹配度未知**：不清楚自身背景与岗位要求的差距
- **缺乏反馈闭环**：投递结果无法转化为改进策略

### 1.2 用户画像

- **目标用户**：2027届硕士毕业生，国际中文教育专业
- **求职目标**：AI产品经理/相关实习岗位
- **技术背景**：具备Claude API、Prompt Engineering等技能
- **核心需求**：快速评估匹配度，生成针对性简历

## 二、方案设计

### 2.1 系统架构

```
用户层（Gemini风格前端）
    ↓
服务层（FastAPI后端）
    ├─ 简历解析器（LLM驱动）
    ├─ JD解析器（LLM驱动）
    ├─ 匹配引擎（向量+关键词）
    ├─ 简历生成器（LLM重写）
    └─ 投递追踪（反馈闭环）
    ↓
数据层（SQLite）
```

### 2.2 核心功能

| 模块 | 功能 | 技术实现 |
|------|------|----------|
| 简历解析 | PDF/DOCX/文本 → 结构化数据 | LLM提取 |
| JD解析 | 岗位描述 → 能力标签 | LLM提取 |
| 匹配计算 | 简历×JD → 匹配度分数 | 关键词覆盖率 + 向量相似度 |
| 原子库 | 拆解简历为可重用原子 | 数据库存储 |
| 简历生成 | 针对JD重写简历 | LLM优化描述 |
| 投递追踪 | 记录结果→优化权重 | 数据反馈 |

### 2.3 创新点

**经历原子库**：将简历"乐高化"，按需重组

- 原始模式：固定简历 → 所有岗位
- 新模式：经历原子库 → 针对每个岗位定制生成

**反馈闭环**：投递结果反向优化关键词权重

- 记录每次投递使用的关键词
- 标记面试/拒信结果
- 高转化率关键词自动提升权重

## 三、AI工具选型

### 3.1 LLM选择：Kimi (月之暗面)

| 维度 | 选择理由 |
|------|----------|
| 成本 | 免费额度充足，适合学生场景 |
| 中文能力 | 国产模型，中文理解优秀 |
| API稳定 | 已验证litellm兼容性 |
| 响应速度 | 平均2-3秒，体验流畅 |

### 3.2 数据存储：SQLite

| 选择 | 理由 |
|------|------|
| SQLite | 本地运行，零成本，适合当前单机原型 |
| WAL 模式 | 比 JSON 文件存储更适合并发读写 |

### 3.3 后端框架：FastAPI

| 优势 | 说明 |
|------|------|
| 异步支持 | 高并发场景性能优 |
| 自动文档 | Swagger UI开箱即用 |
| 类型验证 | Pydantic减少bug |

### 3.4 前端方案：静态HTML + Gemini风格设计

| 选择 | 理由 |
|------|------|
| 静态HTML | 无需构建，直接部署 |
| Gemini风格 | 高级简约，符合产品定位 |
| Tailwind CDN | 快速开发，样式统一 |

## 四、关键配置

### 4.1 环境变量 (.env)

```bash
# Kimi API
KIMI_API_BASE=https://api.moonshot.cn/v1
KIMI_API_KEY=sk-xxxxx

# 模型配置
LLM_MODEL=kimi
DEFAULT_MODEL=moonshot-v1-8k
```

### 4.2 匹配权重配置

```python
weights = {
    "hard_skills": 0.40,    # 硬技能 40%
    "soft_skills": 0.20,   # 软技能 20%
    "education": 0.15,     # 教育 15%
    "experience": 0.15,    # 经验 15%
    "projects": 0.10       # 项目 10%
}
```

### 4.3 数据库结构

```text
backend/data/offer_catcher.db
├── resumes       # 简历记录
├── jobs          # 岗位记录
├── matches       # 匹配历史
├── atoms         # 经历原子库
└── applications  # 投递追踪
```

## 六、部署方案

### 6.1 当前部署（本地）

```bash
# 一键启动
./start.sh

# 或分别启动
cd backend
uvicorn app.main:app --reload --port 8888

cd ../frontend
python3 -m http.server 3000
```

访问地址：
- 前端：http://localhost:3000/index.html
- 后端：http://localhost:8888
- API 文档：http://localhost:8888/docs

### 6.2 公网部署方案（零成本）

| 组件 | 方案 | 成本 |
